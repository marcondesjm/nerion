Instagram Auto Publisher

Procedimento para Automatizar Outra Conta do Instagram

Guia prático para reaproveitar o template/plugin criado neste projeto em uma nova conta, com validação de acentos, prévia visual, dry-run e publicação via Instagram Graph API.

Projeto: NerionOS
Arquivo base: plugins/instagram-auto-publisher
Atualizado em: 2026-07-15

1. Visão Geral

O fluxo reutilizável permite criar carrosséis automáticos para outras contas do Instagram usando configuração por conta, packs de conteúdo, estilos visuais alternáveis e publicação via GitHub Actions.

O processo tem quatro fases:

  1. Preparar a conta no Meta/Instagram.
  2. Criar a configuração da conta no template.
  3. Validar e renderizar uma prévia sem publicar.
  4. Rodar dry-run e depois publicar/agendar.
Comece sempre por --validate-copy e --render-only. Só depois rode dry-run. Publicação real vem por último.

2. Pré-Requisitos

  • Conta Instagram profissional ou criador de conteúdo.
  • Conta vinculada a uma Página do Facebook no Meta Business.
  • Acesso ao Meta Developers / Graph API para gerar token.
  • INSTAGRAM_USER_ID da conta correta.
  • Token com permissão para criar e publicar mídia.
  • Chave do imgBB ou outro serviço público de hospedagem de imagem.
  • Repositório com GitHub Actions habilitado.

3. Arquivos do Template

Arquivo Função
accounts.example.json Modelo de conta, usuário esperado, marca e nomes dos secrets.
content-packs.example.json Slides e legendas que serão alternados por data.
visual-styles.example.json Cores e estilos visuais do carrossel.
publish-carousel.mjs Motor de validação, renderização, upload e publicação.
instagram-feed-template.yml Workflow base para GitHub Actions.

4. Copiar o Template

O template está dentro do plugin: plugins/instagram-auto-publisher/assets/instagram-template.

Copie para uma pasta de automação da nova conta:

automation/instagram-template

Ou use um nome por cliente:

automation/instagram-cliente-x
Não copie a pasta runs/. Ela contém prévias e logs gerados localmente.

5. Configurar a Conta

Duplique accounts.example.json para accounts.json e adicione a conta: 

{
  "account": "cliente-x",
  "expectedUsername": "cliente.oficial",
  "brandName": "Cliente X",
  "footerText": "Automação e crescimento com método",
  "accessTokenEnv": "CLIENTE_X_INSTAGRAM_ACCESS_TOKEN",
  "userIdEnv": "CLIENTE_X_INSTAGRAM_USER_ID",
  "imgbbKeyEnv": "IMGBB_API_KEY",
  "scheduleUtc": "10 12 * * *"
}

O campo expectedUsername é uma trava de segurança: o script consulta a Graph API e falha se a conta retornada não for exatamente essa.

6. Criar Conteúdo

Duplique content-packs.example.json para content-packs.json. Cada pack representa um carrossel completo. 

{
  "account": "cliente-x",
  "packs": [
    {
      "slides": [
        {
          "eyebrow": "Diagnóstico",
          "title": "Processo claro vende melhor.",
          "body": "Quando o time entende prioridade, mensagem e próximo passo, a operação fica mais leve."
        }
      ],
      "caption": "Processo claro vende melhor.\n\nAutomação boa começa pelo fluxo certo."
    }
  ]
}
O script bloqueia mojibake e palavras comuns sem acento no texto principal. Hashtags podem ficar sem acento quando isso for intencional.

7. Configurar Estilo Visual

Duplique visual-styles.example.json para visual-styles.json. Cada estilo altera cor de destaque, fundo, grade e texto. 

{
  "name": "mint-grid",
  "accent": "#7CFFB2",
  "accentSoft": "rgba(124,255,178,0.18)",
  "grid": "rgba(124,255,178,0.10)",
  "bgTop": "#0B1014",
  "bgBottom": "#121A20",
  "text": "#F4F7F5",
  "muted": "#DDE5DF"
}

8. Configurar Secrets no GitHub

No repositório, vá em:

Settings -> Secrets and variables -> Actions -> New repository secret

Crie os secrets com os nomes definidos em accounts.json:

  • CLIENTE_X_INSTAGRAM_ACCESS_TOKEN
  • CLIENTE_X_INSTAGRAM_USER_ID
  • IMGBB_API_KEY
Não coloque token real em arquivo .json, .env versionado ou no PDF. Use GitHub Secrets.

9. Comandos de Validação Local

Validar textos

node automation\instagram-template\scripts\publish-carousel.mjs --account cliente-x --validate-copy

Gerar prévia sem publicar

node automation\instagram-template\scripts\publish-carousel.mjs --account cliente-x --render-only

Dry-run

node automation\instagram-template\scripts\publish-carousel.mjs --account cliente-x --dry-run

Publicar de verdade

node automation\instagram-template\scripts\publish-carousel.mjs --account cliente-x

10. Configurar GitHub Actions

Copie instagram-feed-template.yml para .github/workflows/instagram-feed-cliente-x.yml.

Defina o agendamento em UTC. Exemplo: 09:10 em São Paulo = 12:10 UTC.

on:
  schedule:
    - cron: "10 12 * * *"
  workflow_dispatch:
    inputs:
      account:
        required: true
        default: "cliente-x"
      dry_run:
        required: true
        default: "true"

No primeiro disparo manual, deixe dry_run=true. Só troque para false depois de validar a conta, imagens e legenda.

12. Como Replicar Para Mais Contas

  1. Adicione uma nova entrada em accounts.json.
  2. Adicione um grupo com o mesmo account em content-packs.json.
  3. Crie secrets próprios para a nova conta.
  4. Crie um workflow por conta ou use um workflow manual com input account.
  5. Teste com --validate-copy, --render-only e dry-run.

13. Problemas Comuns

Erro Causa provável Ação
Conta errada INSTAGRAM_USER_ID aponta para outra conta. Corrigir o user ID e manter expectedUsername como trava.
Texto sem acento Pack foi escrito em ASCII ou gerado sem revisão. Corrigir texto e rodar --validate-copy.
imgBB upload failed Chave ausente, inválida ou limite do serviço. Verificar IMGBB_API_KEY.
Graph POST failed Token sem permissão, expirado ou mídia recusada. Renovar token e revisar permissões da Meta Graph API.
